Born and raised in India, been working on this project for some time now. These days anyone can start a personal site or a blog but learning and creating from the basics adds up to your knowledge. What's New?

Let's start with the basics these days having an online presence is very crucial. With the ever-growing population first names (.com) will most of the time be out of the market or be very expensive to buy. Owning your first and second name (.com) comes to the next best in the current scenario. Which I already own and is live with a WordPress blog on it.

Next in line is the first name in other tlds (.co) or the (.xyz) which are also in my possession, now I just need to put all these domains to work. I don't want to just forward the domain name to a single blog or any other social profile page. So, I am just creating this static site using plain simple HTML and CSS for this one.

What's the Plan?

My plan is simply to create one blog on WordPress (ShaanChopra.com), I have HTML site on (ShaanChopra.net) and I am going to use (Shaan.xyz) as a test site. Now the two leftovers (Shaan.co/Shaan.me) are failovers and going to act as a buffer when required.

Some people will call me "CRAZY" for doing this but with the tech development these days if you are not current then you are junk. I can not keep jumping IPs or changing content management systems to and back with my primary domain. That will just put pressure on me and if something bad happens then all is lost. So, keeping this in mind I am going to use buffers and test sites while sailing in uncharted waters.
