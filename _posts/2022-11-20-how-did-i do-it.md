# I did something but what?

I just managed to update my site from VS Code, I don't know what I did, but the site got updated. I really don't like this kind of update where I don't know what I am doing. I also hate installing applications that are beyond my understanding. Just for starting this site, I had to install applications on my laptop that I have no knowledge what they do. I know they are not harmful and are perfectly fine to install them like gem and git from the official sites, but why.

To run a small little document site I should not be installing applications. This is where all the static site generators rolled out. If I have to compare this to WordPress, Ghost and HTML site installation I think this is the toughest. Only a developer can understand what is happening to the site. I need to learn and understand this so that I can have full confidence in this application. 

I am also looking at HUGO for a static site, I think I am going to have the same kind of experience on that platform as well. Deciding between two evils, I don't understand why can't we have ease of use for such a thing. I think there should be an option of filling in the blanks and your site is ready. I am also thinking if I have to write the post in the code form then why not start an HTML site itself?
